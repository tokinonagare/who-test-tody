# 测试人员管理 Bot (Test-Bot) 🤖

这是一个基于 Telegram 的测试人员随机分配与管理机器人。它旨在公平、高效地管理团队测试任务，支持分组管理和基于时间维度的“最久未测优先”分配策略。

## ✨ 核心特性

- **多维度公平分配**：
    - **跨组轮询**：支持将人员分为 6 个组，每次抽取优先从不同组中选择人员，确保测试覆盖面。
    - **最久未测优先**：引入时间戳记忆，即便全员状态重置，也会优先选择最早测试的那批人。
    - **权重机制**：从未参与过测试（无时间戳）的人员拥有 **最高优先级**，会排在队列最前端。
- **自动化管理**：
    - **全员自动重置**：当所有人均完成测试后，自动将状态重置为“未测试”，并开启新的一轮。
    - **状态持久化**：使用 SQLite 数据库存储，支持容器部署，数据永不丢失。
- **灵活配置**：
    - 支持手动设置测试状态、手动调整分组、添加/移除成员。

## 🚀 快速开始

### 1. 获取 Telegram Token
在 Telegram 中私聊 [@BotFather](https://t.me/botfather)，创建一个新 Bot 并获取其 API Token。

### 2. 环境配置
在项目根目录创建 `.env` 文件，并填入您的 Token：
```env
TELEGRAM_TOKEN=你的_TELEGRAM_BOT_TOKEN
```

### 3. 使用 Docker 部署 (推荐)
```bash
docker-compose up -d
```

### 4. 本地部署
```bash
# 安装依赖
pip install -r requirements.txt

# 运行 Bot
python bot.py
```

## 🎮 命令手册

### 基础管理
- `/add <名字>` - 添加新的测试人员（初始状态为未测试、未分组）。
- `/remove <名字>` - 从名单中移除某人。
- `/status` - 查看当前所有人的已测试/未测试状态简报。

### 分组管理
- `/setgroup <名字> <1-6>` - 将人员分配到指定的 1-6 组。
- `/groups` - **[推荐查看方式]** 按组查看人员名单及其测试状态，清晰展示团队分布。

### 历史记录 (New!)
- `/times` - 查看所有人的历史测试时间记录（按从远及近排序）。

### 核心分配
- `/assign <人数>` - **核心命令**。随机抽取指定人数。
    - *逻辑说明*：系统会优先在各组间进行轮询，并从各组中挑选出 **“最久未参与测试”** 的人员。

### 手动调节
- `/set <名字> 1` - 手动标记为 **已测试**，并将测试时间同步更新为当前时间。
- `/set <名字> 0` - 手动标记为 **未测试**，并 **清空** 其测试时间记录（使其在下次抽取时具备最高优先级）。

## 🛠 技术架构
- **语言**: Python 3.10+
- **框架**: `python-telegram-bot`
- **数据库**: SQLite (自动处理字段升级与兼容)
- **部署**: Docker & Docker Compose
